# PRD v8 (최종 확정): 실거주지 추천 서비스

## 개요

사용자의 자산·소득·직장 정보를 입력받아 전월세 최적 주거 옵션을 추천하는 웹 서비스.
국토부 실거래가 공공 API로 시세를 산출하고, 직방 딥링크로 실제 매물 탐색에 연결한다.

> **MVP 범위**: 전월세만 지원. 매매는 UI 비활성화("2차 출시 예정" 툴팁) 처리.

---

## 사용자 플로우

### 재방문 시 — 검색 히스토리 화면

```
최근 검색 기록

[직장: 강남구 테헤란로 / 자산 5,000만원 / 전월세 / 대중교통]  2일 전  [삭제]
[직장: 판교 카카오 / 자산 8,000만원 / 전월세 / 자가용]        1주 전  [삭제]
[직장: 여의도 IFC / 자산 3,000만원 / 전월세 / 대중교통]       2주 전  [삭제]

"+ 새로 검색하기"
```

- 카드 클릭 → 현재 시점 데이터로 재조회 + "최신 시세로 재조회합니다" 토스트 표시
- 개별 삭제 버튼 제공
- 최대 10개 저장, 초과 시 가장 오래된 항목 자동 삭제

---

### Step 1. 기본 정보 입력

- **보유 자산**: 텍스트 입력 + 실시간 포맷팅 (예: 5000 → "5,000만원")
- **월 소득 (세후)**: 동일 방식
- **직장 주소**:
  - 타이핑 중 → 카카오 로컬 API (`/v2/local/search/keyword`) 인라인 드롭다운 자동완성
  - 최종 확정 → 카카오 Daum Postcode API 팝업으로 정확한 주소 + 위경도 확정

---

### Step 2. 조건 설정

- **주거 형태**: 전월세 / ~~매매~~ (매매 Disabled + "2차 출시 예정" 툴팁)
- **선호 주거 유형**: 원룸 / 투룸 / 아파트 / 무관
- **이동 수단**: 대중교통 / 자가용
- "바로 결과 보기" 스킵 버튼 제공 (재방문자용)
- Step 2 선택값 → 결과 필터바 기본값으로 자동 세팅

---

### Step 3. 추천 결과

#### 필터바 (Step 2 선택값이 기본값)

```
[전세만 · 월세만 · 전체]
[원룸 · 투룸 · 아파트 · 무관]
[대중교통 · 자가용]
[출퇴근: 30 / 45 / 60 / 90분]
[대출 포함 ON/OFF]
[정렬: 추천순 · 고정비순 · 출퇴근순]
```

#### 대출 포함 ON 클릭 시

드롭다운으로 대출 상품 선택 → 선택 즉시 가용 자산·결과 재계산

| 상품 | 한도 | 금리 소스 | 비고 |
|---|---|---|---|
| 버팀목 전세자금대출 | min(전세가 × 80%, 2억 2천만원) | HF Open API 실시간 | 소득 5천만원 이하, 전세가 3억 이하 |
| 청년 버팀목 전세자금대출 | min(전세가 × 80%, 2억원) | HF Open API 실시간 | 만 34세 이하, 소득 5천만원 이하 |
| 일반 은행 전세대출 | min(전세가 × 80%, 연소득 × DSR 40% ÷ 금리) | 하드코딩 연 3.8% | UI에 "연 3.8% 기준" 명시 |

> HF API 장애 시 fallback: 버팀목 연 2.7%, 청년 버팀목 연 2.1% 하드코딩값 사용 + "고시 금리 기준" 표시

#### 카드 기본 노출

- 지역명
- 주거 형태 + 가격 요약 (예: 전세 1.8억)
- 최소 필요 자본금
- 월 고정비
- 출퇴근 시간 (선택 이동 수단 기준)
- 추천 점수

#### 카드 클릭 시 확장

- 생활권 요약 (편의시설, 역세권 여부)
- 최근 3개월 실거래가 평균 (데이터 없을 시 "시세 데이터 부족" 뱃지)
- 평수·연식 대비 시세 적정성
- 미니 지도 (해당 위치 중심, 반경 1km 편의시설 핀 표시)
- "직방에서 매물 보기" 버튼 → 직방 딥링크

#### 하단 토글

```
기본값: 목록 뷰
  → "지도로 보기" 클릭 시 전체 지도 뷰 전환
     - 추천 지역 핀 전체 + 클러스터링 적용
     - 핀 클릭 시 말풍선으로 카드 요약 정보 노출
     - 지도 이동 시 재검색 없음 (2차 기능)
  → "목록으로 보기" 클릭 시 카드 리스트 복귀
```

#### 로딩 상태

- 스켈레톤 카드 5개 즉시 표시 → 데이터 도착 순서대로 점진 렌더링
- 3초 내 응답 없을 경우 "데이터를 불러오는 중입니다 (최대 15초)" 안내 표시

## Hard Filter

```
대출 포함 OFF: 전세보증금 or 월세보증금 > 보유 자산 → 결과 제외
대출 포함 ON:  전세보증금 or 월세보증금 > (보유 자산 + 대출 한도) → 결과 제외
```

결과 0건 시:
- "조건 내 결과가 없습니다"
- 대출 포함 OFF 상태면 → "대출 포함 시 N개 결과가 있습니다" CTA 표시

## 월 고정비 계산

```
공통: 대출 필요액 = max(0, 보증금 - 보유 자산)

전세:
  월 고정비 = (대출 필요액 × 선택 상품 금리) ÷ 12 + 관리비

월세:
  월 고정비 = 월세 + (대출 필요액 × 선택 상품 금리) ÷ 12 + 관리비
```

> 관리비 처리 방식은 개발 중 결정 (미결 항목 참고)

## 추천 점수 알고리즘

```
추천 점수 = 출퇴근 점수 (40%) + 가격 적정성 점수 (40%) + 생활권 점수 (20%)
```

### 출퇴근 점수 (40점, 편도 기준)

| 소요 시간 | 점수 |
|---|---|
| 20분 이하 | 40점 |
| 21~30분 | 35점 |
| 31~45분 | 25점 |
| 46~60분 | 15점 |
| 61~90분 | 5점 |
| 90분 초과 | 0점 |

- 대중교통: ODsay API
- 자가용: 카카오모빌리티 자동차 길찾기 API
- API 실패 시: 직선 거리 기반 추정값 사용 + "추정 시간" 표시

**출퇴근 API 호출 최적화**:
1. 직선 거리 컷오프 필터로 1차 후보 압축
2. 상위 10개 먼저 계산, 스크롤 시 추가 lazy 계산

### 가격 적정성 점수 (40점)

```
비교 단위: 전용면적 3.3㎡당 실거래 평균가
점수 = max(0, 40 - (해당 매물가 ÷ 지역 평균가 - 1) × 200)
```

- 최근 3개월 실거래 없을 시: 6개월 → 1년 순으로 확장
- 그래도 없을 시: "시세 데이터 부족" 뱃지 표시, 해당 점수 제외 후 나머지 항목으로 정규화

### 생활권 점수 (20점)

- 검색 반경: 1km 고정
- 카테고리별 점수 및 카운트 상한:

| 카테고리 | 점당 점수 | 최대 카운트 |
|---|---|---|
| 지하철역 (SW8) | 5점 | 3개 |
| 대형마트 (MT1) | 3점 | 3개 |
| 편의점 (CS2) | 2점 | 5개 |
| 병원 (HP8) | 2점 | 5개 |
| 카페 (CE7) | 1점 | 5개 |
| 식당 (FD6) | 1점 | 5개 |

- 정규화: 결과셋 전체 후보 중 최고 합산값을 20점으로 정규화

## API 에러 처리 정책

| API | 실패 시 처리 |
|---|---|
| 국토부 실거래가 | "시세 데이터 부족" 뱃지, 가격 점수 제외 후 나머지로 정규화 |
| ODsay / 카카오모빌리티 | 직선 거리 기반 추정값 + "추정 시간" 표시 |
| 카카오 로컬 (생활권) | 생활권 점수 0점 + "생활권 정보 없음" 표시 |
| HF Open API (금리) | 하드코딩 fallback (버팀목 2.7%, 청년 2.1%) + "고시 금리 기준" 표시 |
| 전체 결과 0건 | "조건 내 결과 없음" + 대출 포함 유도 CTA |

---

## 국토부 실거래가 API 아키텍처

- **1차**: 실시간 직접 호출
- **전환 조건**: 결과 화면 평균 로딩 10초 초과 발생 시 배치 수집으로 전환
- **설계 원칙**: 데이터 소스 추상화 레이어를 두어 전환 시 서비스 코드 변경 최소화
- **배치 전환 시**: Supabase + Vercel Cron, 주 1회 갱신, 캐시 날짜 UI 표시
- **서비스 지역**: 전국 (데이터 없는 지역은 "시세 데이터 부족" 처리)

## 데이터 소스

| 항목 | 소스 | 방식 |
|---|---|---|
| 실거래가 / 시세 | 국토부 실거래가 공공 API | 실시간 (→ 배치 전환 가능) |
| 주소 자동완성 드롭다운 | 카카오 로컬 API `/v2/local/search/keyword` | 실시간 |
| 주소 최종 확정 | 카카오 Daum Postcode API | 팝업 |
| 대중교통 출퇴근 시간 | ODsay 대중교통 API | 실시간 (lazy) |
| 자가용 출퇴근 시간 | 카카오모빌리티 자동차 길찾기 API | 실시간 (lazy) |
| 생활권 점수 | 카카오 로컬 API (카테고리 반경 검색) | 실시간 |
| 지도 표시 | 카카오맵 API | 실시간 |
| 대출 금리 (정책) | HF Open API | 실시간 (캐시 TTL 1일) |
| 대출 금리 (일반 은행) | 하드코딩 연 3.8% | - |
| 실제 매물 연결 | 직방 딥링크 | - |
| 데이터 저장 | localStorage | - |

## 개발 중 결정 항목 (미결 5개)

| # | 항목 | 옵션 |
|---|---|---|
| 1 | 추천 점수에 선호도 반영 | A: Hard Filter만 / B: 일치 시 보너스 점수 |
| 2 | 출퇴근 시간 필터 기본값 | A: 전체 표시 / B: 60분 이하 기본 적용 |
| 3 | 관리비 처리 | A: 고정비 제외 / B: 유형별 하드코딩 (원룸 5만, 아파트 15만 등) |
| 4 | 결과 카드 최대 노출 수 | 20개 / 30개 / 페이지네이션 |
| 5 | 생활권 점수 정규화 | 현재 결과셋 내 상대 정규화로 확정 |
